坦克大战
====
画坦克
------
* 画出两个轮子
* 画出转向器
* 画出圆盘

_子弹逻辑（Version2.0）_
-------
* 每次按J键时调用Tank.shoot()方法，生成一个新的子弹，但是每秒钟只能生成2颗子弹
* 每次生成一颗子弹后，都要在子弹队列中检查该子弹是否还活着，如果没有将子弹从Vector<Bullet> bullets中remove掉。
* 因为添加了子弹不能像原来那样只在按键时调用mp.repaint(),要形成子弹能够运动，就必须每隔一段时间调用mp.repaint()，因此就要将MyPanel类实现Runnable接口，这样就可以使用线程来控制。

_子弹是否击中(Version2.1)_
---------
* 将子弹集合放到MyPanel类中，便于判断是否击中坦克
* 添加击中Tank功能

_爆炸效果(Version 2.2)_
---------------
* 定义类Boom，这个类主要是做爆炸效果绘图，有一个坦克死亡，就生成一个Boom类，然后在图片paint函数中画出来，由于线程每过一段时间画一次，就可以形成动画效果。
* 使用三张图片，快速切换，形成爆炸效果

_敌机移动（Version 3.0）_
---------
* 将敌机实现一个Runnable接口，然后让敌机不停的移动，但是这里和键盘操作Hero移动不同，这里需要使用线程休息函数，因为键盘操作的响应函数自带延迟效果，如果这里不延迟，会造成敌机移动过快的情况

_敌机可发射子弹(Version 3.1)_
--------
* 根据敌机走的步数nstep发射子弹，没有规律，在敌机的run方法中实现。


_英雄死亡与复活机制(Verison 3.2)_
---------------------
* 敌机可击中英雄并且修复子弹生成位置的Bug
